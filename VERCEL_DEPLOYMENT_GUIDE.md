# Vercel Deployment Guide - Golden Star School

This guide provides step-by-step instructions for deploying the updated Golden Star School repository to Vercel. The recent hotfix moved student creation to a secure serverless API and removed insecure admin usage from the frontend.

## Prerequisites

1.  **Vercel Account:** A Vercel account connected to your GitHub.
2.  **Supabase Project:** An active Supabase project with the database schema applied.
3.  **Environment Variables:** You will need your Supabase URL, Anon Key, and Service Role Key.

## Deployment Steps

### 1. Import Project to Vercel

1.  Go to the [Vercel Dashboard](https://vercel.com/dashboard).
2.  Click **"Add New..."** and select **"Project"**.
3.  Find the `the-golden-star-school` repository and click **"Import"**.

### 2. Configure Build Settings

Vercel should automatically detect the project type, but ensure the following settings are correct:

*   **Framework Preset:** Other (or Vite if detected)
*   **Build Command:** `npm run build`
*   **Output Directory:** `dist/public`
*   **Root Directory:** `./` (Leave as default)

### 3. Set Environment Variables

This is the most critical step for the student creation system to work. Add the following environment variables in the Vercel project settings:

| Variable Name | Value | Description |
| :--- | :--- | :--- |
| `VITE_SUPABASE_URL` | `https://your-project.supabase.co` | Your Supabase Project URL |
| `VITE_SUPABASE_ANON_KEY` | `your-anon-key` | Your Supabase Anon/Public Key |
| `SUPABASE_SERVICE_ROLE_KEY` | `your-service-role-key` | **SECRET:** Your Supabase Service Role Key (Server-side only) |

**Note:** The `SUPABASE_SERVICE_ROLE_KEY` is used by the serverless function in `/api/create-student.ts` to create auth users. It is never exposed to the frontend.

### 4. Deploy

1.  Click **"Deploy"**.
2.  Wait for the build and deployment process to complete.
3.  Once finished, Vercel will provide you with a production URL (e.g., `https://the-golden-star-school.vercel.app`).

## Verifying the Student Creation System

After deployment, test the student creation system:

1.  Log in as an **Admin**.
2.  Go to **Manage Students** or **Registration Requests**.
3.  Attempt to create or approve a student.
4.  The system should now call the `/api/create-student` endpoint, which securely creates the user in Supabase Auth and adds the student record to the database.

## Troubleshooting

*   **404 or DEPLOYMENT_NOT_FOUND on /api/create-student:** Ensure the `api/` folder is at the root of your repository and contains `create-student.ts`.
*   **500 Internal Server Error:** Check the Vercel function logs in the "Logs" tab of your Vercel project to see specific error messages from the serverless function. Usually, this is due to missing or incorrect environment variables.
*   **Authentication Errors:** Ensure that the `VITE_SUPABASE_ANON_KEY` is correct, as it is used to validate the admin's session before allowing student creation.
